生成树问题的一些解法

生成树问题的一些解法

一、最小生成树算法

1. kruskal 算法

kruskal 算法是最常用最方便的求解最小生成树算法。

其基本思想为合并连通块，初始时，每个结点自身为一个连通块，然后按边权大小依次尝试加入边使连通块合并，直至只存在一个连通块。

正确性证明：

反证法，假设 kruskal 算法得到的生成树不为原图的最小生成树，那么一定存在一条不在该生成树上的边 (u,v)(u,v)(u,v) 能够替代生成树上 uuu 到 vvv 的路径上权值最大的边，且该边权值大于边 (u,v)(u,v)(u,v) 的权值，那么边 (u,v)(u,v)(u,v) 就会在该边之前加入到生成树中，矛盾，故 kruskal 算法正确性得证。

初始时需将边按边权排序，维护连通块可以使用并查集，故时间复杂度为 O(mlog⁡m)O(m\log m)O(mlogm)，瓶颈在于边权排序。

kruskal 贪心的性质可以很好地解决一些完全图生成树问题。

2. prim 算法

prim 算法与最短路算法中 dijkstra 类似，其基本思想为：从一个连通块出发，每次拓展一个距离当前连通块最近的节点。

如果换个视角，可以发现 prim 算法与 kruskal 算法只是边加入的顺序不同，所以 prim 算法的正确性是正确的。

朴素 prim 算法时间复杂度为 O(n2)O(n^2)O(n2)，时间复杂度瓶颈为查找距当前连通块最近的节点，可以用堆优化，时间复杂度变为 O(mlog⁡m)O(m\log m)O(mlogm)。

prim 算法也是一种利用贪心的算法，可以用于解决一些完全图生成树问题。

3. boruvka 算法

boruvka 的核心思想是合并连通块，每一轮每个连通块选择一个距离最近的联通块然后与其合并，不难发现每轮复杂度为 O(m)O(m)O(m)，每一轮连通块个数都会至少减半，故最多进行 O(log⁡n)O(\log n)O(logn) 轮，总复杂度为 O(mlog⁡n)O(m\log n)O(mlogn)。

次小生成树及类次小生成树问题

[BJWC2010] 严格次小生成树

小 C 最近学了很多最小生成树的算法，Prim 算法、Kruskal 算法、消圈算法等等。正当小 C 洋洋得意之时，小 P 又来泼小 C 冷水了。小 P 说，让小 C 求出一个无向图的次小生成树，而且这个次小生成树还得是严格次小的，也就是说：如果最小生成树选择的边集是 EME_MEM​，严格次小生成树选择的边集是 ESE_SES​，那么需要满足：(value(e)value(e)value(e) 表示边 eee 的权值) ∑e∈EMvalue(e)<∑e∈ESvalue(e)\sum_{e \in E_M}value(e)<\sum_{e \in E_S}value(e)∑e∈EM​​value(e)<∑e∈ES​​value(e)

n,m≤105n,m\le 10^5n,m≤105

可以发现，对于任意一个严格次小最小生成树来说，一定存在一种最小生成树上至多有一条边与该生成树不同。

证明：考虑反证法，若和任意最小生成树均有至少两条边不同，可以考虑割掉其中一条并加入最小生成树上边，生成树边权和将会变小且仍大于最小生成树边权和，与严格次小的定义矛盾。

因此我们可以考虑选择任意一个最小生成树，并枚举不在最小生成树上的边 (u,v)(u,v)(u,v)，加入该边后原图会形成一个环，若环上边权最大值等于 wu,vw_{u,v}wu,v​，删去环上次大的边权即可，否则删除环上最大边权，最后将答案取最小值即为所求。

环上的最大次大边权即为生成树上 (u,v)(u,v)(u,v) 路径上的最大次大边权，用树上倍增或树链剖分维护均可。

这种求解最小生成树的思想也可以解决许多其他题目。

CF827D Best Edge Weight

给定一个点数为 n​n​n​，边数为 m​m​m​，权值不超过 109​10^9​109​ 的带权连通图，没有自环与重边。

现在要求对于每一条边求出，这条边的边权最大为多少时，它还能出现在所有可能的最小生成树上，如果对于任意边权都出现，则输出 −1-1−1。

$ 2 \le n \le 2 \times 10^5,n-1 \le m \le 2 \times 10^5​$

依旧考虑选择一个最小生成树，然后将边分成两类讨论，

对于不在最小生成树上的边 (u,v)​(u,v)​(u,v)​，只有当加入这条边会使得最小生成树更优时才会出现在所有可能的最小生成树中，故其答案即为树上 (u,v)​(u,v)​(u,v)​ 路径上边权的最大值减 1​1​1​。

对于在最小生成树上的边 (u,v)(u,v)(u,v)，只有当加入不在生成树上任意一条边均不会被替代时才会出现在所有可能的最小生成树中，故答案为所有不在最小生成树的边中，加入该边形成的环包含边 (u,v)(u,v)(u,v) 的边的权值最小值加 111，这一步可以在枚举不在最小生成树上的边 (x,y)(x,y)(x,y) 的时候对路径 (x,y)(x,y)(x,y) 上所有边对 wx,yw_{x,y}wx,y​ 取 min⁡\minmin 实现，使用树链剖分和树上倍增均可。

完全图最小生成树问题

CF1242B 0-1 MST

有一张完全图，nnn 个结点

给出 mmm 条边的边权为 111，其余的都为 000，

问你这张图的最小生成树的权值

n,m≤105n,m\le 10^5n,m≤105

考虑 prim 算法的过程，维护一个集合，每一轮拓展一个距离该集合最近的节点。

发现不好维护出这个集合，那么考虑维护出未在集合中的点，直接用数组记录即可。

对每一个新加入集合的点，暴力进行一轮拓展，即枚举每一个当前未在集合中的点，将所有边权为 000 的边的终点加入集合中。

若某一时刻无法拓展，一定需 111 的代价将一个点与当前集合连边，将答案加 111 并加入任意一个点即可。

这个算法的复杂度如何？

不难发现，每次枚举未在集合的点时，有两种情况：

  1. 连接该点的边边权为 000，被拓展。

  2. 连接该点的边边权为 111，被跳过。

对于第一种情况，每次拓展必然减少一个未在集合中的点，故这部分复杂度为 O(n)O(n)O(n)。

对于第二种情况，每次跳过意味着这两点之间有连边，故这部分复杂度为 O(m)O(m)O(m)。

如何快速判断一个点是否与其他点有连边？

枚举该点所有出边，用 visvisvis 数组记录即可，记录时间戳可以避免清空。

这样我们就在严格线性的时间内解决了这个问题，不难发现，这个算法可以输出方案，还可以拓展到边权不为 111 的情况，维护一个堆将加入任意点改为加入据当前集合最近的点即可。

CF1508C Complete the MST

给定一个有 nnn 个节点的完全图，其中 mmm 条边有给定的边权，剩下的没有给定。

你需要给所有没有给定边权的边赋上非负整数边权，使得所有边的边权的异或和等于 000。

我们认为这个图的“丑陋值”为这个图的最小生成树的边权之和，求所有可能的赋值方案中，“丑陋值”的最小值。

由于 a⊕b≤a+ba\oplus b\le a+ba⊕b≤a+b 不难发现，一定存在一种最优解使得只存在一条未给定边权的边边权不为 000，且边权为确定边权边的边权的异或和。

同时若补图有环，一定存在一条补图上的边不在最小生成树上，可以将边权赋给该边，直接套用第一题的做法解法即可。

若补图无环，那么 n(n−1)2−m≤n−1\frac{n(n-1)}2-m\le n-12n(n−1)​−m≤n−1 发现此时点数大概在 m\sqrt mm​ 级别，总边数也在可接受的范围内，那么可以将边都建出来考虑。

先将原图边跑一遍最小生成树，去掉所有不可能出现在最小生成树中的边，然后强制选补图中所有边，然后跑最小生成树，不难发现，此时跑 kruskal，若该边两端点未联通，加入该边，否则可以判断是否用该边替代补图边的边权。

判断补图是否有环可以通过补图连通块个数和补图边数的大小判断，可以套用第一题的做法。
